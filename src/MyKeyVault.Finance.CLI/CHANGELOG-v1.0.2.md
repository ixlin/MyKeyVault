# 更新日志 v1.0.2

## 🔧 本次更新（关键修复）

### 1. **真实性验证股票代码**
**问题**：之前只检查格式（6位数字），但不验证股票是否真实存在。例如输入 `234234` 会被错误接受。

**修复**：
- 输入股票代码后，立即调用 API 验证该股票是否存在
- 查询该股票上一年度的财报数据来验证
- 如果股票不存在，提示用户重新输入
- 示例提示：
  ```
  ❌ 无法查询到股票 234234.SH 的数据，请确认股票代码是否正确。
  提示：请输入沪深A股的6位代码，如 600036（招商银行）、000001（平安银行）。
  ```

### 2. **数据为空检查**
**问题**：即使报表数据为空（如查询未来年份的报表），程序仍然导出"成功"，生成空的 Excel 文件。

**修复**：
- 在获取每张报表后，检查数据是否为空
- 如果任何一张报表数据为空，提示用户原因并回到主菜单
- 不会生成空的 Excel 文件
- 友好提示：
  ```
  ❌ 未查询到 600036.SH 在 2025年年度 的资产负债表数据。
  可能原因：
    1. 该报告期的财报尚未公布
    2. 公司在该期间未上市或已退市
  请重新选择其他年度或报告期。
  ```

### 3. **保留的优化功能（v1.0.1）**
- ✅ 输入验证循环（股票代码、年度、报告期）
- ✅ 友好错误提示，程序不退出
- ✅ 连续导出多只股票
- ✅ 业务主键去重（解决重复记录问题）
- ✅ 优化列标题："报表项目" + "项目值"
